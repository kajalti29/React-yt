<!-- ✅ 1️⃣ What is Git -->
Git is a distributed version control system used to track changes in source code and manage collaboration between developers.

<!-- 🔥 2️⃣ What is Version Control? -->
Version control is a system that tracks changes to files over time so we can recall specific versions later.

<!-- 🔥 3️⃣ Difference Between Git and GitHub? -->
Git is a version control system, while GitHub is a cloud platform that hosts Git repositories.


<!-- 🔥 4️⃣ What is Staging Area? -->
The staging area is a temporary area where changes are prepared before committing.

<!-- 🔥 5️⃣ Explain Git Workflow -->
First we initialize the repository using git init, then add files using git add, commit changes using git commit, and finally push the code to GitHub using git push.
